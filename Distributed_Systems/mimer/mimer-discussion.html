Mimer Discussion

 

                This exercise was a demonstration in creating Back Channel communications from client to server. We created this by creating an asynchronous call to the webserver from our client browser which will in turn have the client talk back to the server in a special protocol. This process lends itself well to unique applications where the client will need to process data, but not want to go specifically through the browser, just initiated by the browser interactions.

                The first section of this assignment was to create the associated type in windows so that Windows knows how to process the file. Put simply we are taking a file and associating it in the windows registry so that Windows knows there is a new data type. After this data type is created we then find the file we would like to execute and attempt to execute it. The execution process will then allow us to associate the file with the shim.bat file downloaded from the assignment.

                After the file type was associated with the shim.bat we needed to create a handler for the data. In this case I will skip over Handler.java and talk about BCHandler.java as this is the process we stitched together. The BCHandler is passed the location of the temporary file that it will process. After that it reads in the file and creates s DataArray class from the data. This DataArray class is then marshalled by XStream into an xml compliant format that will be easier to manipulate the data in the end. This data is then saved locally and send to the server on its backchannel port. When the server receives the XML it unmarshalls the data back into its original DataArray class from the provided XStream functions. This demonstrates how we can serialize entire classes as well as state to another machine in the background away from user interactions.

                Finally the MyWebServer.java file was modified to accept the backchannel communication through the insertion of the provided code snippet. This snippet allows for the server to accept the backchannel message and process it with XStream to get the data passed in the message. This data is then displayed in the server logs.

                In this entire project was a great introduction into how to create a backchannel system that can be used to improve the user experience through unique client server interactions without user interference. This process only took about 3 hours, but provided a very unique way of machine interactions that I did not know much about prior. It also gave insight into ways to transmit classes and state between machines which should be very useful in the future with distributed agents.
